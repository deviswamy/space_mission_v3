# Auth Design ‚Äî Mission Control

**Date:** 2026-03-10
**Status:** Decided

---

## Decision

Use **better-auth** for authentication and session management, with custom fields added directly to its `user` table via `additionalFields`. All user identity data lives in one place. Sessions carry `orgId`, `role`, and `mustChangePassword` ‚Äî no extra queries per request.

---

## Why better-auth over Auth0

- Users stay in our Postgres ‚Äî Drizzle type inference works across all tables
- Multi-tenancy is our own SQL ‚Äî no paid Auth0 Organizations plan
- Session-based by default ‚Äî matches PRD intent
- TypeScript-first, Drizzle adapter ships with the library
- No vendor lock-in, no external service to configure

---

## Schema

better-auth manages four tables via its Drizzle adapter. We extend `user` with three additional fields.

```sql
-- better-auth managed (generated by its Drizzle adapter)
user
  id text PK
  name text
  email text UNIQUE
  emailVerified boolean
  image text
  createdAt timestamptz
  updatedAt timestamptz
  -- our extensions via additionalFields:
  orgId uuid FK ‚Üí orgs.id    NOT NULL
  role  text                  NOT NULL  -- 'director' | 'mission_lead' | 'crew_member'
  mustChangePassword boolean  NOT NULL  DEFAULT true

session
  id text PK
  expiresAt timestamptz
  token text UNIQUE
  userId text FK ‚Üí user.id
  ipAddress text
  userAgent text
  createdAt timestamptz
  updatedAt timestamptz

account
  id text PK
  accountId text
  providerId text
  userId text FK ‚Üí user.id
  accessToken text
  refreshToken text
  ...

verification
  id text PK
  identifier text
  value text
  expiresAt timestamptz
```

Our remaining tables (`orgs`, `skills`, `crew_skills`, `crew_availability`, `missions`, `mission_skill_requirements`, `assignments`) are managed entirely by us.

## better-auth Configuration

```typescript
// server/lib/auth.ts
import { betterAuth } from 'better-auth';
import { drizzleAdapter } from 'better-auth/adapters/drizzle';
import { db } from './db';

export const auth = betterAuth({
  database: drizzleAdapter(db, { provider: 'pg' }),
  emailAndPassword: { enabled: true },
  user: {
    additionalFields: {
      orgId: {
        type: 'string',
        required: true,
        fieldName: 'org_id',
      },
      role: {
        type: 'string',
        required: true,
      },
      mustChangePassword: {
        type: 'boolean',
        required: true,
        defaultValue: true,
        fieldName: 'must_change_password',
      },
    },
  },
});

// Type-safe session user ‚Äî inferred from better-auth config
export type SessionUser = typeof auth.$Infer.Session.user;
// SessionUser includes: id, email, orgId, role, mustChangePassword
```

## Invite Flow

better-auth's built-in invitation system uses email magic links. We bypass it ‚Äî director sets a temporary password directly.

```typescript
// server/routes/admin/invite.ts
import { hash } from '@node-rs/argon2'; // better-auth's default hasher
import { auth } from '../../lib/auth';
import { db } from '../../lib/db';
import { user } from '../../../shared/schema/auth';

export async function inviteUser(req: Request, res: Response) {
  const result = InviteUserSchema.safeParse(req.body);
  if (!result.success) return res.status(400).json({ errors: result.error.issues });

  const { email, role, temporaryPassword } = result.data;
  const { orgId } = req.identity; // set by auth middleware

  const existingUser = await db.select().from(user).where(eq(user.email, email)).limit(1);
  if (existingUser.length > 0) return res.status(400).json({ error: 'Email already in use' });

  const passwordHash = await hash(temporaryPassword);

  await db.insert(user).values({
    id: crypto.randomUUID(),
    email,
    name: email,  // can be updated on profile edit
    emailVerified: true,
    passwordHash,
    orgId,
    role,
    mustChangePassword: true,
    createdAt: new Date(),
    updatedAt: new Date(),
  });

  return res.status(201).json({ success: true });
}
```

## Auth Middleware

Applied to all protected routes. Enriches `req.identity` ‚Äî consumed by every downstream handler.

```typescript
// server/middleware/requireAuth.ts
import { auth } from '../lib/auth';
import type { SessionUser } from '../lib/auth';

declare global {
  namespace Express {
    interface Request {
      identity: SessionUser;
    }
  }
}

export async function requireAuth(req: Request, res: Response, next: NextFunction) {
  const session = await auth.api.getSession({ headers: req.headers });

  if (!session) return res.status(401).json({ error: 'Unauthorized' });

  if (session.user.mustChangePassword && req.path !== '/api/auth/change-password') {
    return res.status(403).json({ error: 'PASSWORD_CHANGE_REQUIRED' });
  }

  req.identity = session.user;
  next();
}

// Role guard factory
export function requireRole(...roles: string[]) {
  return (req: Request, res: Response, next: NextFunction) => {
    if (!roles.includes(req.identity.role)) {
      return res.status(403).json({ error: 'Forbidden' });
    }
    next();
  };
}
```

## Org Isolation

All resource queries include `orgId` from `req.identity`. Cross-org access returns 404.

```typescript
// Pattern used in every resource handler
const mission = await db
  .select()
  .from(missions)
  .where(
    and(
      eq(missions.id, missionId),
      eq(missions.orgId, req.identity.orgId), // enforced here
    )
  )
  .limit(1);

if (!mission.length) return res.status(404).json({ error: 'Not found' });
```

**Wouter auth guard:**

```typescript
// client/src/lib/AuthGuard.tsx
export function AuthGuard({ children }: { children: React.ReactNode }) {
  const { data: session, isPending } = useSession();
  const [, navigate] = useLocation();

  useEffect(() => {
    if (!isPending && !session) navigate('/login');
    if (session?.user.mustChangePassword) navigate('/change-password');
  }, [session, isPending]);

  if (isPending || !session) return null;
  return <>{children}</>;
}
```

## Rejected Alternatives

| Option | Reason rejected |
| --- | --- |
| Auth0 | Users stored externally (breaks Drizzle typing), Organizations is paid, JWT-first |
| Custom sessions (express-session + bcrypt) | More boilerplate, no type inference on session, session store setup needed |
| better-auth + parallel user_profiles table | Extra join on every request, session object doesn't carry orgId/role, split identity |
| better-auth organization plugin | Diverges from our schema design, overengineered for 2 seeded orgs |
